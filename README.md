1. Откройте https://github.com/new

**В новый пустой репозиторий загрузите файлы в таком порядке:**

1. Нажмите **"uploading an existing file"**
2. Перетащите **ВСЕ** файлы из папки `csr-survey/`:
   ```
   ├── index.html
   ├── vercel.json
   ├── requirements.txt
   ├── .gitignore
   ├── api/
   │   ├── stats.py
   │   └── download.py
   └── README.md (этот файл)
   ```

3. **ВАЖНО:** `index.html` должен быть **в корне** репозитория!
4. Нажмите **"Commit changes"**

## 📋 ШАГ 2: ДЕПЛОЙ В VERCEL

### 2.1 Удалите старый проект (если есть)

1. Откройте https://vercel.com/dashboard
2. Найдите проект `csr-survey-fixed` (старый)
3. Settings → General → Delete Project
4. Подтвердите удаление

### 2.2 Создайте новый проект

1. Vercel Dashboard → **"Add New..."** → **"Project"**
2. Нажмите **"Import Git Repository"**
3. Найдите репозиторий **`csr-survey`**
4. Нажмите **"Import"**

### 2.3 Настройте проект

**На странице настройки проекта:**

1. **Project Name:** `csr-survey` (точно так!)
2. **Framework Preset:** `Other` ← **КРИТИЧЕСКИ ВАЖНО!**
3. **Root Directory:** оставьте пустым или `.`
4. **Build Command:** оставьте пустым
5. **Output Directory:** оставьте пустым
6. **Install Command:** оставьте пустым

### 2.4 Деплой

1. Нажмите **"Deploy"**
2. Подождите 1-2 минуты
3. Вы увидите: ✅ **Congratulations!**

## 📋 ШАГ 4: ЕСЛИ URL НЕ `csr-survey.vercel.app`

Если Vercel дал другой URL (например, `csr-survey-abc123.vercel.app`):

### 4.1 Проверьте имя проекта

1. Vercel Dashboard → ваш проект
2. Settings → General
3. **Project Name** должно быть: `csr-survey`
4. Если другое - измените на `csr-survey`
5. Save

### 4.2 Добавьте правильный домен

1. Settings → Domains
2. Нажмите **"Add"**
3. Введите: `csr-survey.vercel.app`
4. Если занято, попробуйте:
   - `enpf-survey.vercel.app`
   - `csr-monitoring.vercel.app`
   - `pension-survey.vercel.app`

### 4.3 Обновите index.html

Если URL изменился:

1. В GitHub откройте `index.html`
2. Найдите строку:
   ```javascript
   const API_BASE = 'https://csr-survey.vercel.app';
   ```
3. Замените на ваш реальный URL
4. Commit changes

## 🆘 TROUBLESHOOTING

### Проблема: 404 Not Found

**Причина:** Framework Preset не "Other"

**Решение:**
1. Settings → Build & Development Settings
2. Framework Preset → "Other"
3. Save
4. Deployments → Redeploy (без кэша)

### Проблема: Excel пустой

**Причина:** Нет данных в Kobo

**Решение:** Проверьте что в Kobo есть данные

### Проблема: "Failed to fetch"

**Причина:** Неправильный API_BASE в index.html

**Решение:** Проверьте что URL совпадает с доменом в Vercel
